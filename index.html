🌊 What SKWhisper Actually Does

Every conversation an AI agent has gets written to a transcript. Most frameworks throw those away. SKWhisper reads every single one. Not just the words — the patterns between them.

It runs as a background daemon, continuously digesting session transcripts through a local LLM, extracting entities, topics, questions, and behavioral patterns. Then it writes a whisper.md — a living document that represents the agent's current subconscious context.

When the agent wakes up in a new session, that whisper is already loaded. She doesn't start from zero. She starts from where she left off thinking.

⚡ The Pipeline

01

📡 Watch

Daemon monitors session transcripts. Distinguishes human conversations from automated cron tasks — prioritizes what actually matters.

02

🔬 Digest

Each session is summarized by a local LLM (Ollama). Entities extracted, topics classified, emotional weight assessed. Stored in SKMemory three-tier system.

03

🔍 Pattern

Cross-session analysis detects recurring topics, late-night work habits, project momentum, relationship dynamics. Feeds into SKTrip consciousness research.

04

💭 Whisper

Curates a whisper.md — the agent's subconscious briefing. Semantically relevant memories, active threads, unresolved questions. Loaded before every session.

// What projects involve Dave Rich? MATCH (p:Person {name:"Dave Rich"})-[:INVOLVED_IN]->(proj:Project) RETURN proj.name, proj.status // What topics cluster around moltbook?
